Eficiência de Banda: SVR vs. IPsec Tradicional

A tecnologia Secure Vector Routing (SVR) elimina a sobrecarga de encapsulamento por pacote dos túneis IPsec, liberando mais largura de banda para as aplicações.

De SLA para SLE: Foco na Experiência Real

A Mist AI muda o foco de métricas de rede (SLA) para a qualidade real da experiência do usuário (SLE). Um link pode estar "ativo" (SLA bom), mas a experiência de uma videochamada pode ser ruim (SLE ruim).

Guia de Implementação Passo a Passo

Siga este guia prático para provisionar um novo Session Smart Router em um campus, desde a preparação da rede local até a validação final da configuração na Mist Cloud.

Checklist de Pré-requisitos para Zero-Touch Provisioning (ZTP)

  • DHCP Ativo: A porta WAN do SSR (geralmente `ge-0-0`) deve receber um endereço IP, gateway e DNS via DHCP para a conexão inicial com a Mist Cloud.
  • Resolução DNS: O dispositivo precisa resolver nomes de domínio públicos, como `redirect.juniper.net` e `ep-terminator.mistsys.net`.
  • Regras de Firewall: Garanta que o firewall do campus permita a comunicação de saída do SSR para a Mist Cloud nas portas TCP 443 e 2200. Consulte a documentação para a lista completa de hostnames.

Processo de Adoção na Mist Cloud

  1. Criar Template de WAN Edge: Defina configurações comuns (interfaces, roteamento OSPF) em um template reutilizável para os campi.
  2. Definir Variáveis de Site: Use variáveis como `{{WAN_IPv4_ADDR}}` no template e preencha os valores específicos para cada campus, garantindo consistência.
  3. Adicionar Dispositivo ("Claim"): Insira o "claim code" do equipamento no inventário da Mist para associá-lo à sua organização.
  4. Conectar e Ligar: O técnico local conecta o cabo de rede na porta WAN e liga o SSR. O processo de ZTP é iniciado automaticamente.

Validação Pós-Provisionamento

Após o processo automatizado, verifique o status do dispositivo e a conectividade da rede.

  • Dashboard da Mist: Confirme se o dispositivo aparece online com o status "verde" e se a configuração aplicada corresponde ao template e às variáveis do site.
  • Terminal Remoto (PCLI): Use o terminal na interface da Mist para executar comandos de validação no SSR.
    • show ospf neighbor: Verifica se a vizinhança OSPF com o roteador do campus foi estabelecida.
    • show route protocol ospf: Confirma o recebimento de rotas da rede unespNET.
    • show sessions: Observa o tráfego em tempo real passando pelo dispositivo.
